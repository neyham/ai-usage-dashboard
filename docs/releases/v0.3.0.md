## AI Usage Dashboard v0.3.0

OpenAI Build Week release candidate extending the existing local-first Windows
dashboard with provider selection and a credential-free Judge Demo.

### Highlights

- Select any combination of Codex, Claude, and DeepSeek for the home screen.
- Disabled providers are excluded from credential reads and network refreshes.
- Selections persist atomically, while disabled-provider caches remain available
  for a later re-enable.
- Zero, one, two, and three-panel layouts are covered across Surface landscape,
  portrait, Snap, touch, and compact windows.
- The **AI Usage Dashboard (Judge Demo)** Start menu shortcut launches embedded
  synthetic fixtures without reading live configuration, credentials, or cache.

### Install

Use `AI-Usage-Dashboard_0.3.0_x64-setup.exe` for the Build Week judging path.
The installer is not code-signed, so Windows SmartScreen may require explicit
confirmation. Verify the final public download against its published SHA-256
before running it.

### Build Week scope

The provider-selection workflow, Judge Demo, expanded layout tests, submission
draft, and 0.3.0 packaging were completed on 2026-07-17 with Codex and GPT-5.6.
The provider integrations, Tauri architecture, cache, Surface UI, and v0.2.0
release existed before the event.

### Compatibility note

Live mode reads credential formats and usage endpoints used by provider CLI
tools. Some are undocumented and may change without notice. Judge Demo has no
dependency on those endpoints.
